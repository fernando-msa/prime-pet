# Release v2026.4.2

Tipo: stable
Tag: v2026.4.2
Data: 2026-04-20

## Resumo

Release focada em fechamento de fluxo ponta a ponta com hardening de segurança, validação operacional em Windows e estabilização de toolchain da API v2.

## Mudanças incluídas

### Produto e documentação

- Inclusão no README da clareza de proposta em nível de produto (problema, público e uso real).
- Inclusão de galeria visual no README com screenshots e GIF da aplicação em execução.
- Atualização do README com comandos validados para API v2 no PowerShell (`npm.cmd`).

### Segurança e hardening

- Endurecimento de regras de acesso em agendamentos no Firestore com ownership por `ownerUid`.
- Restrição de update de slot para admin e delete condicional para owner em status pendente.
- Validação forte de URL de webhook no admin (somente HTTPS público; bloqueio de localhost e redes internas).
- Mitigação de XSS no dashboard com renderização via `textContent` para dados dinâmicos.
- Documento de evidências e checklist em `docs/planning/SECURITY_HARDENING_2026-04.md`.

### Qualidade e validação

- Script de smoke QA para Windows em `scripts/qa_smoke.ps1`.
- Smoke QA validado em PowerShell e também via Git Bash.
- API v2 validada com lint e build após ajustes.

### Dependências e vulnerabilidades

- Upgrade de `bcrypt` para `^6.0.0` removendo cadeia vulnerável `@mapbox/node-pre-gyp -> tar`.
- Alinhamento de `typescript` para `^5.5.4`, eliminando warning de compatibilidade no lint.
- Auditoria de dependências da API v2 finalizada sem vulnerabilidades (`npm audit`).
