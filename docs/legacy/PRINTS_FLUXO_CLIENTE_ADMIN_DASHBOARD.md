# Prints do fluxo: Cliente / Admin / Dashboard

Abaixo estão os prints de referência do fluxo operacional para documentação e alinhamento com produto/operação.

## 1) Cliente (`client.html`)

![Fluxo do cliente](../../assets/screenshots/fluxo-cliente.svg)

Foco do print:

- criação de pedido;
- acompanhamento de status;
- visibilidade da jornada até confirmação.

## 2) Admin (`admin.html`)

![Fluxo do admin](../../assets/screenshots/fluxo-admin.svg)

Foco do print:

- lista de pendências;
- decisão operacional (confirmar/cancelar);
- status de integração via webhook.

## 3) Dashboard (`dashboard.html`)

![Fluxo do dashboard](../../assets/screenshots/fluxo-dashboard.svg)

Foco do print:

- KPIs principais;
- funil de conversão;
- saúde operacional (SLA e integrações).

## Uso recomendado

- Referenciar estes prints no processo de release.
- Atualizar imagens quando houver mudança relevante de UX.
- Anexar versão mais recente em issues de melhoria do fluxo.
